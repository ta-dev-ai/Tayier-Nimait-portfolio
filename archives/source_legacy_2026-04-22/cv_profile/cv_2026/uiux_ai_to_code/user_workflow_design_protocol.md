# User Workflow Design Protocol

This document captures the new method added to the UI/UX archive.

## Main idea

Before generating the image design, the system must understand:

- the user workflow
- the actions of the user
- the business goal
- the data type used
- the output expected

## Design order

1. Audit workflow
2. Verify missing features
3. Define the design scope
4. Generate the dashboard image
5. Convert the image to code constraints
6. Implement the code

## Why this matters

This avoids hallucination and keeps the UI tied to real usage.

## Example checkpoints

- import Excel or CSV data
- convert history to JSON
- visualize with a bar chart
- calculate VAT and totals
- show status and analytics
