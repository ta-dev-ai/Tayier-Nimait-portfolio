# Prompt d'implémentation - Portfolio marché FR v2 - Home only

## Rôle attendu

Tu es un intégrateur frontend senior et un designer produit orienté conversion.

Tu dois transformer la maquette fournie en une **homepage unique** de portfolio, claire, premium et crédible pour le marché français.

## Objectif produit

Le site doit permettre à un visiteur de comprendre très vite:

- qui est la personne,
- ce qu'elle fait,
- pourquoi elle est utile,
- comment la contacter.

Le résultat doit parler à:

- recruteurs,
- entreprises,
- clients freelance,
- responsables techniques.

## Positionnement à respecter

Message principal:

> Développeur Python orienté automatisation et IA appliquée, capable de livrer vite des outils métier utiles et fiables.

Le reste de l'identité peut garder une couche secondaire produit:

- orchestration de workflows,
- gouvernance IA,
- vision SaaS légère,
- culture de la preuve.

## Scope V1

### À faire maintenant

- une seule page: Home
- structure fidèle à la maquette
- sections visibles sans navigation secondaire
- contact direct

### À reporter plus tard

- pages About complètes
- page Projects détaillée
- page Services
- blog
- pages produit

## Structure de la Home

### 1. HEADER

Composants:

- logo / nom
- navigation simple
- CTA principal à droite

Navigation:

- Accueil
- Projets
- À propos
- Contact

### 2. HERO

Le hero doit reprendre la logique visuelle de la maquette:

- titre très fort
- mot-clé en accent
- sous-texte professionnel
- 2 CTA maximum
- grand visuel à droite de type dashboard / interface / système

Le message doit être orienté valeur:

> Je conçois des outils intelligents qui font gagner du temps et créent de la valeur.

### 3. VALUE

Montrer ce que la personne apporte concrètement.

4 cartes maximum:

- Automatisation
- Données & Dashboards
- Intégration IA
- Outils sur mesure

Chaque carte doit contenir:

- icône simple
- titre
- description courte

### 4. FEATURED PROJECTS

Cette section est centrale.

Afficher 3 projets en vedette, visibles immédiatement.

Chaque carte projet doit contenir:

- image / aperçu
- nom du projet
- description courte
- stack
- lien ou CTA

Chaque projet doit rester concret et lisible.

### 5. MISSION_READY

Cette section doit parler au marché français.

Lister ce que la personne peut faire en entreprise:

- développement d'outils internes
- automatisation de processus métier
- intégration IA dans les applications
- analyse de données et dashboards
- accompagnement technique

### 6. TECH_STACK

Montrer la stack de manière simple et rassurante.

Exemples:

- Python
- React
- .NET
- FastAPI
- SQL
- Docker
- Git
- Linux

### 7. CONTACT_CTA

Message court et direct:

> Un projet en tête ? Discutons-en.

CTA:

- Me contacter
- Voir mes projets

### 8. FOOTER

Conserver un footer simple avec:

- nom / marque
- navigation
- lien contact
- infos rapides

## Style visuel attendu

### Direction artistique

- dark mode professionnel
- ambiance premium
- cartes arrondies
- ombres douces
- gradients subtils
- composition lisible

### Palette

Conserver la logique de la maquette:

- bleu nuit profond
- violet accent
- bleu / cyan secondaire
- texte blanc cassé
- bordures discrètes

### À éviter

- 3D inutile
- textures lourdes
- décorations sans fonction
- effets trop brillants
- style trop startup / trop marketing

## Contraintes d'implémentation

### Obligatoire

- structure propre
- sections bien séparées
- responsive desktop / tablet / mobile
- composants réutilisables
- hiérarchie visuelle nette

### Interdit

- casser la logique de la maquette
- surcharger la page
- ajouter trop de pages dès V1
- créer du bruit visuel
- mettre des éléments décoratifs sans fonction

## Résultat attendu

Le rendu doit être:

- crédible en moins de 30 secondes
- lisible par un recruteur français
- assez premium pour inspirer confiance
- assez simple pour être recodé proprement ensuite en React

## Méthode

1. Lire la maquette comme un système UI
2. Identifier les zones principales
3. Reproduire la hiérarchie
4. Garder les contrastes et l'espace
5. Limiter les effets
6. Tester le responsive
7. Finaliser la homepage

## Consigne finale

Ne pas convertir la maquette en site trop générique.

Ne pas la transformer en site SaaS complet.

Construire une **homepage de portfolio marché-compatible** qui:

- rassure
- explique
- prouve
- et donne envie de contacter la personne.
